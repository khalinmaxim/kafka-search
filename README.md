# Kafka Search

Веб-приложение для поиска сообщений в Kafka без коммита офсетов и без влияния на консюмеров.

## Зачем

Стандартные Kafka UI (Kowl, Redpanda Console и др.) плохо справляются с поиском по содержимому сообщений — они делают full scan без индексов, часто падают по таймауту на больших топиках. Это приложение решает ту же задачу, но с нормальным UX: фильтрация по JSON-полю, поиск по подстроке, прогресс сканирования.

**Данные в Kafka не изменяются** — приложение только читает, офсеты не коммитит, consumer group не создаёт.

## Запуск

```bash
git clone ...
cd kafka-search
go run .
```

Открыть в браузере: http://localhost:8080

Требуется Go 1.21+.

## Использование

### 1. Подключение

**Brokers** — адрес брокера(ов) через запятую:
```
broker1:9092,broker2:9092
```

**Address overrides** — если Kafka возвращает внутренние адреса брокеров, недоступные с твоей машины, укажи маппинг `внутренний=внешний`:
```
internal-broker:9092=accessible-broker:9092
```

Нажми **Load Topics** — список топиков загрузится в дропдаун с поиском.

### 2. Поиск

| Поле | Описание |
|------|----------|
| **Search Query** | Что искать |
| **Field** | JSON-поле для точного совпадения. Поддерживает вложенность: `data.user_id`. Если не указано — ищет подстроку в сыром тексте |
| **From** | Читать с этого момента. Пусто = с начала топика |
| **Limit** | Максимум найденных сообщений (не сканированных) |

### Примеры

Найти все сообщения где `id == 1567`:
- Query: `1567`
- Field: `id`

Найти по подстроке в любом месте сообщения:
- Query: `some-string`
- Field: *(пусто)*

Найти по вложенному полю `payload.source_id == 42`:
- Query: `42`
- Field: `payload.source_id`

### 3. Результаты

- Клик по строке — раскрывает JSON с подсветкой синтаксиса
- Статус-бар показывает: сколько найдено / сколько просканировано / из скольких партиций
- Если часть партиций недоступна (лидер на другом брокере) — показывается предупреждение

## Ограничения

- Читает только с партиций, лидер которых доступен. Если лидер партиции — недоступный брокер, эта партиция пропускается с предупреждением.
- Сообщения только в JSON. Поиск по полю на бинарных форматах (Avro, Protobuf) не работает.
- Нет аутентификации (SASL/SSL не поддерживается в текущей версии).
